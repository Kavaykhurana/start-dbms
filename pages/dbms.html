This is a Startup Investment Analysis and Decision System.
The frontend shows a dashboard, startup portfolio, recommendation engine, analytics, and DBMS documentation.
The backend exposes API endpoints that read normalized SQL data.

Trigger: Equity Check

Blocks a new investment if active equity for a startup crosses 100%.

CREATE OR REPLACE TRIGGER trig_EquityCheck
FOR INSERT OR UPDATE OF equity_percentage, startup_id, status
ON investments
COMPOUND TRIGGER
    TYPE t_startup_ids IS TABLE OF investments.startup_id%TYPE INDEX BY PLS_INTEGER;
    g_startup_ids t_startup_ids;
    g_count PLS_INTEGER := 0;

    BEFORE EACH ROW IS
    BEGIN
        g_count := g_count + 1;
        g_startup_ids(g_count) := :NEW.startup_id;
    END BEFORE EACH ROW;

    AFTER STATEMENT IS
        v_total_equity NUMBER(7,2);
    BEGIN
        FOR idx IN 1 .. g_count LOOP
            SELECT NVL(SUM(equity_percentage), 0)
            INTO v_total_equity
            FROM investments
            WHERE startup_id = g_startup_ids(idx)
              AND status = 'Active';

            IF v_total_equity > 100 THEN
                RAISE_APPLICATION_ERROR(-20001, 'Equity cannot exceed 100 percent.');
            END IF;
        END LOOP;
    END AFTER STATEMENT;
END trig_EquityCheck;

Procedure: ROI

Calculates ROI from latest net profit and investor equity.

CREATE OR REPLACE PROCEDURE sp_CalculateROI (
    p_investment_id IN NUMBER,
    p_roi_percentage OUT NUMBER
) AS
BEGIN
    SELECT ROUND(
        ((fm.net_profit_inr * 12 * 5 * (i.equity_percentage / 100))
        / NULLIF(i.invested_amount_inr, 0)) * 100,
        2
    )
    INTO p_roi_percentage
    FROM investments i
    JOIN financial_metrics fm
      ON fm.startup_id = i.startup_id
    WHERE i.investment_id = p_investment_id
      AND fm.metric_month = (
          SELECT MAX(metric_month)
          FROM financial_metrics
          WHERE startup_id = i.startup_id
      );
END;

Complex Join Demo

Use this to explain how investor, startup, sector, investment, financial, and risk tables combine.

SELECT
    inv.investor_name,
    s.startup_name,
    sec.sector_name,
    i.funding_round,
    ROUND(fm.ltv_inr / NULLIF(fm.cac_inr, 0), 2) AS ltv_cac_ratio,
    ra.risk_score
FROM startups s
JOIN sectors sec ON sec.sector_id = s.sector_id
JOIN investments i ON i.startup_id = s.startup_id
JOIN investors inv ON inv.investor_id = i.investor_id
JOIN financial_metrics fm ON fm.startup_id = s.startup_id
LEFT JOIN risk_analysis ra ON ra.investment_id = i.investment_id
WHERE sec.sector_name = 'Fintech'
  AND fm.metric_month = (
      SELECT MAX(metric_month)
      FROM financial_metrics
      WHERE startup_id = s.startup_id
  )
ORDER BY ltv_cac_ratio DESC, ra.risk_score ASC;

Normalization Made Simple

1NFEvery cell stores one atomic value.
2NFMetrics depend on the complete metric row, not part of a combined key.
3NFSector, investor, startup, and metric data are separated to remove transitive dependency.
BCNFUnique determinants like startup_name and sector_name are candidate keys.
